Subject: Advance Web Application

TITLE: Kafé Lumière Milk Tea Shop Management System
DESCRIPTION:
The Kafé Lumière Milk Tea Shop Management System is designed to help manage the daily operations of Kafé Lumière, a milk tea shop. The system serves three types of users: Customers, Cashiers, and Admins. Customers can browse available products, place orders, and view their order history. Cashiers can process customer orders, record payments, and generate receipts. Admins can manage users, products, sales records, and system reports.
The system automatically calculates order totals and generates receipts for each transaction. It also keeps records of sales that can be used to create reports for monitoring shop performance. Only authorized users can access the system, ensuring security and proper data handling.

FUNCTIONAL REQUIREMENTS:
USER MANAGEMENT
• The system shall allow customers to register and create their own accounts.
• The system shall allow customers, cashiers, and admins to securely log in and log out.
• The system shall allow admins to create, edit, and delete cashier and admin accounts.
• The system shall allow admins to manage customer accounts.
• The system shall restrict access based on user roles (Admin, Cashier, Customer).
• The system shall provide account management and password update features.

CUSTOMER MANAGEMENT
• The system shall store customer account information.
• The system shall allow customers to update their profile information.
• The system shall maintain customer transaction history.
• The system shall allow customers to view their previous orders and receipts.

PRODUCT MANAGEMENT
• The system shall allow admins to add, update, and delete milk tea products.
• The system shall manage product categories (Classic, Fruit Tea, Premium, and others).
• The system shall manage add-ons (Pearls, Pudding, Cheese Foam, and others).
• The system shall manage size options and corresponding prices.
• The system shall allow admins to update product availability.

ORDER MANAGEMENT
• The system shall allow customers to place orders through the system.
• The system shall allow cashiers to create and process customer orders.
• The system shall allow selection of drinks, sizes, sugar levels, ice levels, and add-ons.
• The system shall automatically calculate the total price of each order.
• The system shall generate a unique order number for every transaction.
• The system shall allow customers to view the status of their orders.

PAYMENT AND RECEIPT
• The system shall record payment details for each transaction.
• The system shall support cash payment transactions.
• The system shall automatically calculate change when applicable.
• The system shall generate receipts for completed transactions.
• The system shall store transaction history for future reference.

SALES AND REPORTS
• The system shall generate daily sales reports.
• The system shall generate weekly sales reports.
• The system shall generate monthly sales reports.
• The system shall display total revenue and total number of transactions.
• The system shall display best-selling products.
• The system shall allow admins to export reports in PDF or Excel format.

SYSTEM SECURITY
• The system shall encrypt user passwords before storing them in the database.
• The system shall prevent unauthorized access through authentication and role-based
permissions.
• The system shall log user activities such as login, order creation, payment processing, and
product updates.
• The system shall automatically terminate inactive sessions after a specified period.

BACKUP AND DATA MANAGEMENT
• The system shall allow database backup and recovery.
• The system shall securely store all records in the MySQL database.
• The system shall maintain transaction records for reporting and auditing purposes.
• The system shall ensure data integrity and consistency.
